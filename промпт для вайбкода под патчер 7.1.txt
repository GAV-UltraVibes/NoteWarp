Промпт: Разметка кода маркерами [имя>
Формат маркеров
Каждый логический блок кода ОБЯЗАН иметь пару маркеров — открывающий и закрывающий. Между ними находится тело блока.

Открывающий маркер: <синтаксис-комментария> [имя>

Закрывающий маркер: <синтаксис-комментария> <имя]

Синтаксис комментариев по языкам:

Язык	Пример
JavaScript	// [class-app> ... // <class-app]
CSS	/* [styles-header> ... /* <styles-header] */
HTML	<!-- [html-tab> ... <!-- <html-tab] -->
Правила
Уникальность — имена маркеров должны быть уникальны в пределах файла

Вложенность — допускается, но закрывающие маркеры идут в обратном порядке

Однострочные блоки — маркеры могут быть на одной строке с кодом

Соглашения об именах
Префикс	Назначение	Пример
class-*	JavaScript-классы	class-texteditor
method-*	Методы классов	method-undo
styles-*	CSS-стили	styles-header
html-*	HTML-разметка	html-editor-tab
init-*	Инициализация	init-domcontentloaded
event-*	Обработчики	event-keydown
Примеры разметки
JavaScript (класс)
javascript
// [class-texteditor>
class TextEditor {
  // [method-constructor>
  constructor(element) {
    this.element = element;
  }
  // <method-constructor]

  // [method-undo>
  undo() {
    this.history.undo();
  }
  // <method-undo]
}
// <class-texteditor]
CSS
css
/* [styles-header> */
.app-header {
  text-align: center;
  margin-bottom: 20px;
}
/* <styles-header] */
HTML
html
<!-- [html-editor-tab> -->
<div id="editor-tab" class="active">
  <textarea id="editor"></textarea>
</div>
<!-- <html-editor-tab] -->
Формат патча для PatchMaster v7.1
json
[patch>
{
  "meta": {
    "name": "Название патча"
  },
  "verify": {
    "mustContain": ["[class-target>"],
    "mustNotContain": ["версия: \"1.8.0\""]
  },
  "changes": [
    {
      "type": "replace",
      "marker": "имя-маркера",
      "with": "новый код"
    }
  ]
}
<patch]
Поля патча
Поле	Обязательно	Описание
meta.name	✅	Название патча
verify.mustContain	❌	Строки, которые ДОЛЖНЫ быть в тексте
verify.mustNotContain	❌	Строки, которых НЕ ДОЛЖНО быть
changes[].type	✅	Всегда "replace"
changes[].marker	✅	Имя маркера (без скобок и >/<)
changes[].with	✅	Новый код блока
Пример патча
json
[patch>
{
  "meta": {
    "name": "Добавить подсказку при undo"
  },
  "verify": {
    "mustContain": ["[method-undo>"],
    "mustNotContain": ["showHint('↩ Отмена')"]
  },
  "changes": [
    {
      "type": "replace",
      "marker": "method-undo",
      "with": "\n  undo() {\n    const state = this.historyManager.undo();\n    if (state) {\n      this.editor.value = state.val;\n      this.setCursorPosition(state.cur);\n      this.showHint('↩ Отмена', 1000);\n    }\n  }\n  "
    }
  ]
}
<patch]
Логика работы PatchMaster v7.1
Пользователь вставляет патч в текст редактора

Нажимает «🔧 Применить патч» (или Ctrl+Shift+P)

PatchMaster находит маркеры [patch>...<patch]

Парсит JSON внутри маркеров

Проверяет verify (если есть)

Показывает диалог подтверждения

Для каждого изменения:

Находит открывающий маркер [имя>

Находит закрывающий маркер <имя] после открывающего

Заменяет содержимое между ними

Сохраняет отступы от строки с открывающим маркером

Удаляет [patch>...<patch] из текста

Показывает результат

Важно:

Маркеры ищутся как точные вхождения [имя> и <имя]

Комментарии //, /*, <!-- перед маркерами игнорируются

Если блок не найден — патч не применяется, ошибка в лог

Лог доступен 60 секунд через кнопку «📋 Лог»

Полный пример размеченного файла
javascript
// [class-app>
class App {
  // [method-constructor>
  constructor() {
    this.data = [];
    this.init();
  }
  // <method-constructor]

  // [method-init>
  init() {
    this.loadData();
    this.render();
  }
  // <method-init]
}
// <class-app]

// [init-domready>
document.addEventListener('DOMContentLoaded', () => {
  window.app = new App();
});
// <init-domready]
